
question ===> 1

1. Interface ব্যবহার করা হয় মূলত কোনো অবজেক্টের গঠন কেমন হবে সেটা নির্দিষ্ট করতে।
Type যদিও সেটা করতে পারে, কিন্তু এটি শুধু অবজেক্ট নয়—union, primitive, tuple সব কিছুর জন্য ব্যবহার করা যায়।


2. Interface সহজে বাড়ানো বা সম্প্রসারণ করা যায়

Interface এর উপরে আবার নতুন প্রপার্টি যোগ করে বাড়ানো যায় (extends দিয়ে)।
Type দিয়ে এটা করা যায়, কিন্তু interface-এর মতো এতটা সরল ও সুবিধাজনক নয়।


3. একই নামে একাধিক interface একত্র হয়ে যায়

যদি একই নামে দুইটি interface লিখে ফেলো, TypeScript সেগুলোকে একসাথে merge করে ফেলে।
কিন্তু একই নামে দুইটি type লিখলে error দেখায়—কারণ type কখনো merge হয় না।



4. জটিল টাইপ তৈরি করতে Type বেশি সুবিধাজনক

Union type বা conditional type এর মতো জটিল টাইপ চাইলে type অনেক শক্তিশালী।
Interface এসব জটিল কাজ করতে পারে না—এটা শুধু অবজেক্টের কাঠামো নিয়েই কাজ করে।


question ===> 2


1. any – সবকিছুই চলে, কোনো নিয়ম মানতে হয় না

any মানে হচ্ছে—“তোমার যা ইচ্ছা তাই করো।
এই টাইপ দিলে TypeScript আর কোনো চেকই করে না।
ভ্যালু কী টাইপের, কী রিটার্ন করবে, কোথায় ব্যবহার হচ্ছে—কিছুই ধরে না।


2. unknown – ভ্যালুটা আছে, কিন্তু আগে চেক করতে হবে

unknown হলো নিরাপদ any.
এখানে TypeScript বলে:
“ভ্যালু আছে ঠিকই কিন্তু ব্যবহার করার আগে আমাকে জানাও এটা আসলে কী টাইপ।”


3. never – এমন কিছু যা কখনোই ঘটবে না

never মানে—এই ভ্যালু কখনোই আসবে না।
ধরো, এমন একটা ফাংশন যেটা চিরদিন লুপ করে বা সরাসরি error ছুঁড়ে দেয়—ওটা কিছুই রিটার্ন করে না।
সেই ক্ষেত্রে never ব্যবহার হয়।
